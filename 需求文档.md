# 支票大写金额转换器 - 需求文档

## 一、项目概述

### 1.1 项目名称
支票大写金额转换器（Number to Traditional Chinese Cheque）

### 1.2 项目目标
开发一个网页应用，实现阿拉伯数字到繁体中文支票大写金额的转换功能，帮助用户快速准确地填写支票。

### 1.3 目标用户
- 财务人员
- 会计人员
- 需要填写支票的商务人士
- 学习中文数字大写的学生

---

## 二、功能需求

### 2.1 核心功能

#### 2.1.1 数字转换
- **输入**：阿拉伯数字（支持整数和小数）
- **输出**：繁体中文大写金额
- **转换方式**：实时转换（输入即转换）

#### 2.1.2 支持范围
- 整数部分：0 - 99,999（十万以内，不含十万）
- 小数部分：最多支持两位小数（角、分）
- 最小值：0
- 不支持负数
- 超过99,999时显示错误提示

### 2.2 辅助功能

#### 2.2.1 复制功能
- 提供"复制"按钮
- 一键复制转换结果到剪贴板
- 复制成功后显示确认提示

#### 2.2.2 示例展示
- 展示常见的转换示例
- 点击示例可自动填充到输入框
- 至少提供 4 个典型示例

#### 2.2.3 输入验证
- 实时验证输入格式
- 不符合格式时显示错误提示
- 支持输入格式：
  - 纯数字：`100`
  - 带小数：`12345.67`
  - 带逗号分隔：`1,234.56`（自动去除）
- 验证规则：
  - 超过99999时显示："超过最大支持金额（99999.99）"
  - 负数时显示："不支持负数"
  - 非法字符时显示："请输入有效的数字"

## 三、转换规则详细说明

### 3.1 基本数字映射

| 阿拉伯数字 | 繁体中文 |
|-----------|---------|
| 0 | 零 |
| 1 | 壹 |
| 2 | 贰 |
| 3 | 參 |
| 4 | 肆 |
| 5 | 伍 |
| 6 | 陆 |
| 7 | 柒 |
| 8 | 捌 |
| 9 | 玖 |

### 3.2 单位映射

#### 3.2.1 小单位
- 十位：拾
- 百位：佰
- 千位：仟

#### 3.2.2 大单位
- 万：萬（最大单位）

#### 3.2.3 货币单位
- 元：圓
- 角：角
- 分：分
- 整数：整

### 3.3 零的处理规则

#### 规则1：中间有零需要读出
- `1001` → `壹仟零壹圓整`
- `10086` → `壹萬零捌拾陸圓整`

#### 规则2：连续多个零只读一个
- `1000` → `壹仟圓整`（不是"壹仟零零零圓整"）
- `10001` → `壹萬零壹圓整`（不是"壹萬零零零壹圓整"）

#### 规则3：整千、整万后面的零不读
- `1000` → `壹仟圓整`
- `10000` → `壹萬圓整`

#### 规则4：小数部分的零
- 角为零但分不为零：`100.05` → `壹佰圓零伍分`
- 角不为零分为零：`100.10` → `壹佰圓壹角`
- 角分都不为零：`100.12` → `壹佰圓壹角贰分`

### 3.4 特殊情况处理

#### 情况1：整数（无小数部分）
- 末尾加"整"
- `100` → `壹佰圓整`
- `12345` → `壹萬贰仟參佰肆拾伍圓整`

#### 情况2：有小数部分
- 不加"整"，直接写角分
- `100.50` → `壹佰圓伍角`
- `100.55` → `壹佰圓伍角伍分`

#### 情况3：零元
- `0` → `零圓整`
- `0.50` → 零圓伍角`

#### 情况4：十位数特殊处理
- `10` → `壹拾圓整`（不能省略"壹"）
- `20` → `贰拾圓整`

## 四、转换示例

### 4.1 整数示例

| 0.01 | 零圓零壹分 | 只有分 |

| 0.50 | 零圓伍角 | 五角 |

### 4.3 边界情况

| 输入 | 输出 | 说明 |
|------|------|------|
| 99999.99 | 玖萬玖仟玖佰玖拾玖圓玖角玖分 | 最大值 |
| 0.01 | 零圓零壹分 | 最小值 |
| 100000 | 错误：超过最大支持金额（99999.99） | 超出范围 |
